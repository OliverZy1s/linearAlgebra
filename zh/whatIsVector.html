# 向量是什么

# 不同专业视角下的向量

  • 在物理专业的同学眼里，向量是一个由长度和方向定义的箭头（有向线段），只要以上两个特征相同，可以自由移动一个向量而保持它不变， 无论是在二维平面还是三维空间里。

  • 在计算机专业同学的眼里，向量则是一组有序的数字列表。在这种情况下， 向量通常是列表的一种花哨的说法。

  • 而数学专业的同学，则尝试统一这两种说法，只需要保证两个向量相加与数字与向量相乘有意义即可。 这种看待向量的观点有些抽象。不过到本节的最后你肯定能明白。

# 向量的定义

# 二维里的情况

那么向量到底是一个有向线段还是一组有序的数字列表呢？答案是：都是。

我们推荐首先把它想成一个在x-y平面里的有向线段， 而这个有向线段的起点是固定在原点的。同时可以用有向线段的终点坐标表示向量。坐标中的第一个数表示了在x轴方向上移动的距离， 第二个数表示了在y轴方向上移动的距离。每一个向量对应唯一的二元数组。

补充

这可能与物理专业的同学想的不太一样，通常在物理里，向量的起点可以任意变化。 但在线性代数里，向量的起点通常是固定的，因为这样归一化了向量的含义。

# 三维里的情况

在三维里的情况也是一样的，我们推荐首先把它想成一个在x-y-z空间里的有向线段， 而这个有向线段的起点是固定在原点的。同时可以用向量的终点坐标表示向量。坐标中的第一个数表示了在x轴方向上移动的距离， 第二个数表示了在y轴方向上移动的距离，第三个数表示了在z轴方向移动的距离。每个向量对应唯一的三元数组。

# 向量加法与数乘运算

在给出了向量有向线段（几何角度）和坐标（代数角度）这种很自然的定义之后，我们接着来讨论之前提到的加法运算与数乘运算。

# 向量加法运算

# 几何上加法的表示

我们有向量 v⃗\vec{v}​v​⃗​​ 和 w⃗\vec{w}​w​⃗​​，把 w⃗\vec{w}​w​⃗​​ 的起点移到 v⃗\vec{v}​v​⃗​​ 的终点，这样新得到的向量就被定义为 v⃗+w⃗\vec{v}+\vec{w}​v​⃗​​+​w​⃗​​。

补充

我们在之前提到说线性代数里一般向量的尾部都固定在原点，这可能是唯一一种尾部离开原点的情况。

那么如果我们抛开已有的认知，想一下为什么这样定义，而不是其他的？

我们可以把每个向量看作进行一种“动作”，而两个向量相加就是这两个“动作“的和。不妨看看我们是如何定义数字的加法的， 2+5其实就是向右移动两个单位加上向右移动五个单位。而这最终总的结果就是向右移动七个单位。

# 代数上加法的表示

同时，从坐标角度看，向量的定义也很自然。我们可以先沿着第一个向量移动，首先往右移一个单位，然后往上移两个单位， 再沿着第二个向量移动，往右移三个单位，再往下移一个单位。

同时我们也可以两个向量横坐标的作用和纵坐标的作用统一看，也就是向右移（3+1）个单位，向上移（2+（-1））个单位。

总的来说，在“向量是有序的数字列表”的观点里，向量加法就是把对应项相加。

# 向量数乘运算

# 几何上数乘的表示

另一种向量的基本运算是数乘，它的意思就是向量乘上一个常数。在几何上很好理解，就是原有的向量拉伸或者压缩。 比如 2v⃗2\vec{v}2​v​⃗​​ 代表的意思就是将原有的向量拉伸为原来的两倍。

补充

乘上一个负数也是一样的道理，只不过最终效果是使向量反向。

# 代数上数乘的表示

同时在坐标角度上数乘的表示也很清晰，我们只需要将向量的每个坐标都乘上给定的常数 （我们把那个乘上的常数叫做标量）。

# 小结

在之后的章节中你会越来越明白为什么会说线性代数就是围绕向量加法和向量数乘这两种基础运算。 除此之外，我们再回到开始的问题，不同专业视角下的向量。实际上无论怎么看待向量都没有关系。

  • 把向量看成一个有向线段，
  • 把向量看成有序的数字列表。

因为线性代数的效用很少体现在单一一种表示上，而是更多的在这两种观点的相互转换，这恰恰也是线性代数的美丽之处。
